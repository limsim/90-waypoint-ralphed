## Codebase Patterns
- This repo was "ralphed": commit 29db285 removed `index.html` and `src/**` so the app is rebuilt
  story-by-story. The PRD (`scripts/ralph/prd.json`) drives the order; everything starts `passes:false`.
- Canonical PRD + progress live in `scripts/ralph/` (that's what `ralph.sh` reads). A duplicate
  `prd.json` exists at the repo root — ignore it; update the one under `scripts/ralph/`.
- `ralph.sh` does NOT manage git branches. The agent must `git checkout -b <branchName>` from main itself
  (PRD `branchName` is `ralph/90-waypoint-map`).
- Quality gate: `npm run build` (tsc -b), `npm test` (tsc -b && node --test glob), `npm run typecheck`.
  All three must be green before committing.
- `tsc -b` does NOT delete orphaned `dist/` outputs. After removing/renaming a `.ts` file, `rm -rf dist`
  before `npm test` or stale `dist/tests/*.test.js` get re-run by the glob (this bit US-001).
- zsh gotcha: `rm -rf dist *.tsbuildinfo` aborts with "no matches found" (buildinfo lives inside dist/).
  Use plain `rm -rf dist`.
- Layering enforced by project-reference tsconfigs: `core` (domain+application, NO DOM lib) <- `adapters`
  (+DOM, includes src/main.ts) and `test` (+node types). Relative imports need a `.js` extension.

  malformed `%ZZ` (decodeURIComponent throws) -> 403, directory request `/scripts/` -> 404 (no listing).
  All correct — the traversal guard genuinely holds.
- Refinement made: serve.mjs now sends `Cache-Control: no-store` on 200 responses. The `npm run dev`
  (`tsc -b --watch`) + `npm run serve` loop rebuilds dist/ on every save; without this the browser could
  serve a stale `dist/src/*.js` from cache. Squarely within US-001's "serve" criterion and de-risks every
  future UI story (US-013+). Verified header present on both index.html and dist/src/main.js. No build/test impact.
- Files changed: scripts/serve.mjs (no-store header + comment), CLAUDE.md (serve note: no-store + traversal
  guard behaviour). Committed CLAUDE.md + serve.mjs only; left scripts/ralph/ralph.sh uncommitted (runner config).
- **Learnings for future iterations:**
  - To exercise serve.mjs's path-traversal 403 guard you must percent-encode the slashes (`%2f`) too —
    curl/browsers normalise a raw `../` client-side, so a plain `../../etc/passwd` request reaches the
    server already collapsed (becomes a 404 inside ROOT, never hits the 403 path).
  - serve.mjs serves with `Cache-Control: no-store`; when verifying UI stories in the browser a hard
    refresh is unnecessary — reloads always re-fetch fresh compiled modules.
