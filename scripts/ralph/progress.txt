## Codebase Patterns
- This repo was "ralphed": commit 29db285 removed `index.html` and `src/**` so the app is rebuilt
  story-by-story. The PRD (`scripts/ralph/prd.json`) drives the order; everything starts `passes:false`.
- Canonical PRD + progress live in `scripts/ralph/` (that's what `ralph.sh` reads). Duplicate
  `prd.json` AND `progress.txt` have appeared at the repo root — a stray run wrote there because the
  agent resolved bare `progress.txt`/`prd.json` from the repo-root CWD. ALWAYS read/write the copies
  under `scripts/ralph/`; ignore/remove the root duplicates (US-002 progress was lost this way once).
- Point and Bounds are pure value objects in `src/domain/` (no DOM/Canvas); `Bounds.grow(factor)`
  expands/shrinks from the bounding-box centre so the centroid stays stable during generation-space scaling.
- `ralph.sh` does NOT manage git branches. The agent must `git checkout -b <branchName>` from main itself
  (PRD `branchName` is `ralph/90-waypoint-map`).
- Quality gate: `npm run build` (tsc -b), `npm test` (tsc -b && node --test glob), `npm run typecheck`.
  All three must be green before committing.
- `tsc -b` does NOT delete orphaned `dist/` outputs. After removing/renaming a `.ts` file, `rm -rf dist`
  before `npm test` or stale `dist/tests/*.test.js` get re-run by the glob (this bit US-001).
- zsh gotcha: `rm -rf dist *.tsbuildinfo` aborts with "no matches found" (buildinfo lives inside dist/).
  Use plain `rm -rf dist`.
- Layering enforced by project-reference tsconfigs: `core` (domain+application, NO DOM lib) <- `adapters`
  (+DOM, includes src/main.ts) and `test` (+node types). Relative imports need a `.js` extension.

- **Learnings for future iterations:**
  - To exercise serve.mjs's path-traversal 403 guard you must percent-encode the slashes (`%2f`) too —

- **Learnings for future iterations:**
  - Heading.apply() uses a static ALL array cycle; static fields initialise top-to-bottom so ALL must be
